변경된 파일을 분석하여 코드 리뷰를 수행합니다.

## 리뷰 절차

1. `git diff HEAD` 및 `git status`로 변경된 파일과 변경 내용을 확인하세요.
2. 변경된 각 파일을 읽고 아래 두 가지 관점에서 리뷰합니다.

---

## 리뷰 기준

### 1. 코드 퀄리티 (frontend-fundamentals.com/code-quality 기준)

**가독성 (Readability)**
- [ ] **맥락 줄이기** — 같이 실행되지 않는 코드가 분리되어 있는가? 구현 세부사항이 적절히 추상화되어 있는가? 역할이 다른 로직이 하나의 함수에 뭉쳐 있지 않은가?
- [ ] **이름 붙이기** — 복잡한 조건식에 의미 있는 이름이 붙어 있는가? 매직 넘버가 상수로 추출되어 있는가?
- [ ] **위에서 아래로 읽히기** — 코드를 읽을 때 시점이 위아래로 자주 이동하지 않는가? 삼항 연산자가 지나치게 중첩되어 있지 않은가? 조건의 좌항과 우항이 자연스럽게 읽히는가?

**예측 가능성 (Predictability)**
- [ ] **이름 일관성** — 같은 개념에 다른 이름이 사용되거나, 다른 개념에 같은 이름이 겹치지 않는가?
- [ ] **반환 타입 일관성** — 동일한 역할을 하는 함수들이 반환 타입을 통일하고 있는가?
- [ ] **숨은 로직** — 함수 이름만 보고 예측할 수 없는 사이드 이펙트나 동작이 숨어 있지 않는가?

**응집도 (Cohesion)**
- [ ] **파일 구조** — 함께 변경되는 코드가 같은 위치에 있는가?
- [ ] **폼 응집도** — 폼과 관련된 상태, 검증 로직, 핸들러가 한 곳에 모여 있는가?

**결합도 (Coupling)**
- [ ] **단일 책임** — 하나의 함수/컴포넌트가 하나의 책임만 지고 있는가?
- [ ] **Props Drilling** — 불필요하게 여러 단계를 거쳐 props가 내려가고 있지 않는가?
- [ ] **중복 코드** — 비슷해 보이지만 다른 맥락의 코드를 무리하게 추상화하지 않았는가?

---

### 2. 접근성 (Accessibility)

- [ ] **시맨틱 HTML** — 의미에 맞는 HTML 요소를 사용하고 있는가? (`button`, `nav`, `main`, `header` 등)
- [ ] **키보드 접근성** — 모든 인터랙션 요소가 키보드(Tab, Enter, Space)로 접근 및 조작 가능한가?
- [ ] **포커스 관리** — `:focus-visible` 스타일이 적용되어 있는가? 모달·다이얼로그 등 동적 요소에서 포커스가 올바르게 이동하는가?
- [ ] **대체 텍스트** — 이미지에 `alt` 속성이 있는가? 아이콘 버튼에 `aria-label`이 있는가?
- [ ] **ARIA 속성** — `role`, `aria-label`, `aria-live`, `aria-expanded` 등이 적절히 사용되고 있는가? 불필요한 ARIA가 남용되지 않았는가?
- [ ] **색상 대비** — 텍스트와 배경의 명도 대비가 충분한가? (WCAG AA 기준: 일반 텍스트 4.5:1, 큰 텍스트 3:1)
- [ ] **폼 레이블** — 모든 입력 요소에 `label`이 연결되어 있는가? (`for`/`htmlFor` 또는 `aria-label`)
- [ ] **에러 메시지** — 폼 에러가 `role="alert"` 또는 `aria-live`로 스크린리더에 전달되는가?
- [ ] **비활성 상태** — `disabled` 또는 `aria-disabled`가 올바르게 적용되어 있는가?

## 리뷰 출력 형식

리뷰 결과는 아래 형식으로 작성합니다.

```
## 코드 리뷰

### 변경 파일
- 변경된 파일 목록

---

### 코드 퀄리티

#### 잘된 점
- ...

#### 개선이 필요한 부분
| 파일 | 위치 | 문제 | 개선 방안 |
|------|------|------|-----------|
| ...  | ...  | ...  | ...       |

---

### 접근성

#### 잘된 점
- ...

#### 개선이 필요한 부분
| 파일 | 위치 | 문제 | 개선 방안 |
|------|------|------|-----------|
| ...  | ...  | ...  | ...       |

---

### 종합 평가
코드 퀄리티: X / 5
접근성: X / 5
총평: ...
```

변경 사항이 없으면 "리뷰할 변경 파일이 없습니다."라고 알려주세요.
